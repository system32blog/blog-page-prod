PowerShell based PRTG Sensor Discovery

Nov 8, 2021 · 22 min read · Robin Beismann  ·
Share on:

This article shows a script I made to enhance PRTGs own sensor discovery. The script can be run as scheduled tasks and will use the PrtgApi to find any kind of sensors that you wish to monitor on your systems (such as Windows Services, NetApp Volumes, Cisco UCS Disks and so on)

Disclaimer: This content and scripts are provided free of charge and I do not take any responsability for them nor any harm they may cause, it is up to you to review and test them properly before deploying them anywhere.

Introduction

Many of you probably already heard about PRTG or are already using it to monitor your network. PRTG has a great builtin sensor discovery that will add sensors for almost everything you'd want to know about on your server. One downside of this is the fact, that everytime you run this discovery, it adds all sensors once again as it does not check if this sensor is already there.

To workaround this, I've build a PowerShell script that you can run as scheduled job to automatically discover any kind of discoverable sensor.

The script is also able to send you a summary of what it discovered and created directly into your Teams Channel (via an Webhook Connector).

How does it work?

The script uses PRTGs inbuilt API and the PowerShell Module "PrtgAPI" (https://github.com/lordmilko/PrtgAPI) to invoke a sensor discovery for a specific sensor on a device or a group of devices. After that it loops through the existing sensors to detect if there are any that are not explicitly excluded or already existent.

The script has the following features and attributes per sensor type:

  • RawType: The PRTG Name of a sensor, you can find all your PRTG Instance is aware off using the cmdlet Get-SensorType.
  • SensorPrefix: The prefix that is used by the sensor when you add them using the PRTG API.
  • Group: The PRTG Device Group which the script shall run for this sensor type. If multiple groups with the same name exist (e.g. under different probes), it will run for all of them.
  • NameFilter: A wildcard aware filter of devices the script shall run for. Do all your Windows Servers starts with "WSRV"? Great! Enter "WSRV*" into the NameFilter and you'll avoid unnecessary discoveries on not applicable devices. If not, just let the asterix in.
  • AttributeName: The attribute name out of which the script shall extract the discovered sensors. This differs per sensor. You can find the correct property by running New-SensorParameters on a device.
  • ExclusionList: An array of occurences to ignore, in the existing sensor types below, you can find some examples. This exclusion list also supports wildcards.
  • MaxSensors: A threshhold of how many sensors to discover per device. Exceeding this limit will fire an error message.
  • DiscoveryInterval: A limitation of how often this Sensor Type discovery shall run, meaning that even if the script is executed multiple times in a row, if the discovery interval is not yet due again, it will skip this sensor type. An example of this would be disk sensors, discovering them all few days will be more than enough. Windows services however might pop up more often and therefor require a more frequent discovery. The last discovery is saved in the current user registry for each device, if you run the script on another device or under an other user context, it will start fresh.
  • NameExtractionSB: An additional scriptblock that's being run against the object t to extract the individual element name out of a device. This might be required for devices that return inconsistent or badly formatted informations around the actual device name. The Cisco UCS Disks Sensor Type shows an example of it.

How to use it

To use the script, you will need to populate the following environment variables before you execute the script:

 1# Required
 2## PRTG server url
 3$env:PRTGUrl = "https://prtg.internaldomain.com"
 4## PRTG Username
 5$env:PRTGUsername = "svc-discovery"
 6## PRTG Password
 7$env:PRTGPassword = "bacon"
 8
 9# Optional
10## A teams webhook to sent a summary of discovered sensors to (if any)
11$env:TeamsURL = "https://yourtenant.webhook.office.com/webhookb2/.."
12## A teams webhook to sent a summary of discovered sensors to (if any) while using VSCode to work on the script
13$env:TeamsStagingURL = "https://yourtenant.webhook.office.com/webhookb2/.."

Depending on the number of items you got in the specified groups, it will take some time (minutes to hours) to discover the Sensor Types defined below.

Predefined Sensor Types

There is already a bunch of sensor types defined in the script, feel free to either keep them or delete them out of the hashtable. Those are the ones I've developed the script for and are actively used in my work environment.

  • Cisco UCS Disks: Discover disk of a Cisco UCSC and Cisco UCSB Series using their CIMC Interface
  • NetApp System Health: Discover NetApp System Health Sensors for each Controller
  • NetApp IO: Discover NetApp IO Sensors for each Controller
  • NetApp Aggregates: Discover NetApp Aggregates Sensors for all clusters
  • NetApp Lifs: Discover NetApp Lif Sensors for each Controller
  • NetApp Physical Disk: Discover NetApp Disk Sensors for each physical disk
  • NetApp Interfaces: Discover NetApp Physical Interface Sensors for each
  • NetApp Volumes: Discover NetApp Volume Sensors for each Cluster
  • NetApp SnapMirrors: Discover NetApp Snapmirror/SnapVault Relationship Sensors for each Cluster
  • Disk Space: Discover any available SNMP Disk Space Sensor (Windows/Linux) for each device
  • Windows Services: Discover any running Windows Service via SNMP for each device. This Sensor Type already holds a long list of exclusions of builtin Windows Services in English and German OS. The reference lists are from Server 2019 Installations.

573if($env:TERM_PROGRAM -eq 'vscode' -and $env:TeamsStagingURL) {
574    $TeamsURL = $env:TeamsStagingURL
575}else{
576    $TeamsURL = $env:TeamsURL
577}
578#endregion

580#region Functions
581function Send-TeamsMessage($facts,$Theme="Healthy"){
582    $connectorUri = $TeamsURL
583    $Summary = "PRTG Sensor Discovery $(if($env:JOB_NAME){ "(Jenkins Job '$env:JOB_NAME')" } )"
584    $body = [ordered]@{
585        "@type" = "MessageCard"
586        "summary" = $Summary
587        "themeColor" = $(
588            if($Theme.ToLower().Contains("error")){
589                "#eb4034"
590            }else{
591                "#a6a6a6"
592            }
593        )
594        "sections" = @(
595            @{
596                activityTitle = $Summary
597                facts = $facts          
598            }
599        )
600    }
601     
602    Invoke-RestMethod -Method post -ContentType 'Application/Json' -Body ($body | ConvertTo-Json -Depth 5) -Uri $connectorUri
603}
604#endregion
605
606#region Connection and Credentials
607try{
608    # Check if we're connected
609    $null = Get-PrtgStatus
610}catch{
611    # No, let's connect.
612    
613    # Get Credentials
614    $PRTGUsername = $env:PRTGUsername
615    $PRTGPassword = $env:PRTGPassword
616    [securestring]$secStringPassword = ConvertTo-SecureString $PRTGPassword -AsPlainText -Force
617    [pscredential]$creds = New-Object System.Management.Automation.PSCredential ($PRTGUsername, $secStringPassword)
618
619    # Connect
620    $null = Connect-PrtgServer -Server $env:PRTGUrl -Credential $creds -Force
621}
622#endregion
623
624#region Processing
625$addedSensors = @()
626$hasError = @()
627$RegistryBasePath = "Registry::HKEY_CURRENT_USER\SOFTWARE\PRTGDiscovery"
628$RegistryLastDiscovery = "$RegistryBasePath\LastScan"
629
630try{
631    $servicesAdded = @()
632    $SensorTypes.GetEnumerator() | ForEach-Object {
633        try{
634            $SensorType = $_
635            $ignoredServices = $_.Value.ExclusionList.ToLower()
636            
637            # Get Services for the device
638            $services = Get-Device -Group $SensorType.Value.Group | Where-Object { $_.Name -like $SensorType.Value.NameFilter } | ForEach-Object {
639                $device = $_
640                $LastDiscoveryInterval = 4
641
642                # Check if the registry hive is there
643                if(!(Test-Path -Path $RegistryLastDiscovery -ErrorAction SilentlyContinue)){
644                    Write-Host -Object "Creating registry path `"$RegistryLastDiscovery`".."
645                    $null = New-Item -ItemType Directory -Path $RegistryLastDiscovery -Force
646                }
647                $stateName = "$($SensorType.Name) - $($device.Name)"
648
649                # Check if our check is due or if we skip that one
650                if(
651                    (Get-ItemProperty -Path $RegistryLastDiscovery).$stateName -and
652                    ($LastState = Get-ItemPropertyValue -Path $RegistryLastDiscovery -Name $stateName) -and
653                    ($LastStateDate = [datetime]::ParseExact($LastState,'yyyyMMddHHmmss',$null)) -and
654                    ( (Get-Date) -lt $LastStateDate.AddHours($LastDiscoveryInterval) )                
655                ){
656                    Write-Host -Object "[$($device.Name) - $($SensorType.Name)] Skipping as last discovery was `"$LastStateDate`" and it is due at `"$($LastStateDate.AddHours($LastDiscoveryThreshold))`".."
657                    return;
658                }                              
659
660                Write-Host -Object "[$($device.Name) - $($SensorType.Name)] Searching for sensors.."
661                try{
662                    # Null our previous vars out
663                    $sensormeta = $null
664                    
665                    # Build our parameters dynamically for Splatting
666                    $params = @{
667                        RawType = $SensorType.Value.RawType
668                        ErrorAction = 'Stop'
669                    }
670                    if($SensorType.Value.QueryParameters){
671                        $params.QueryParameters = $SensorType.Value.QueryParameters
672                    }
673
674                    # Query the device for SNMP Sensors based on our Template
675                    Set-ItemProperty -Path $RegistryLastDiscovery -Name $stateName -Value (Get-Date -Format "yyyyMMddHHmmss")
676                    $sensormeta = Get-Device -Id $_.Id | New-SensorParameters @params
677                    Write-Host -Object "[$($device.Name) - $($SensorType.Name)] Finished searching for sensors.."
678                }catch{
679                    Write-Verbose -Message "[$($device.Name) - $($SensorType.Name)] Failed to get SNMP sensors: $_"
680                }
681
682                # Filter Services for actual application services
683                $servicesToMonitor = @()
684                if(
685                    $sensormeta -and 
686                    $sensormeta.Targets -and
687                    ($attr = $SensorType.Value.AttributeName) -and
688                    ($sTargets = $sensormeta.Targets.$attr)
689                ){
690                    # Loop through the sensors
691                    $sTargets | Foreach-Object {                        
692                        if(!$SensorType.Value.NameExtractionSB){
693                            $serviceName = $_.Name.Replace("\\","\").ToLower()
694                        }else{
695                            # Cast "t" as var for the device to be used in the scriptblock
696                            $t = $_
697                            
698                            # Invoke the scriptblock
699                            $serviceName = $SensorType.Value.NameExtractionSB.Invoke()
700
701                            # Overwrite it in the hashtable so we can pass the extracted name
702                            $_ | Add-Member -Type NoteProperty -Name "Name" -Value $serviceName -Force
703                        }
704
705                        # Get service name with device name prefix so we can exclude based on specific devices
706                        $serviceNameWithDeviceName = "$($device.Name)\$($serviceName)".Replace("\\","\").ToLower()
707
708                        # Verify if the sensors are not excluded
709                        if(!($ignoredServices | Where-Object {
710                                ($serviceName -like $_) -or
711                                ($serviceNameWithDeviceName -like $_)
712                            }
713                           )
714                        ){
715                            $servicesToMonitor += $_
716                        }
717                    }
718                }else{
719                    # This device has no sensors based on our Sensor Type
720                    Write-Host -ForegroundColor Yellow -Object "[$($device.Name) - $($SensorType.Name)] Has no SNMP Service sensors."
721                }
722
723                # Build the return object 
724                [PSCustomObject]@{
725                    DeviceID = $device.Id
726                    DeviceParentID = $device.ParentId
727                    DeviceName = $device.Name
728                    Services = $servicesToMonitor
729                }
730            }
731            
732            # Look through all detected services and add the sensors
733            $services | Where-Object { $_.Services } | ForEach-Object {
734                # Get the devices
735                $device = Get-Device -Id $_.DeviceId
736
737                # Get the devices' sensors
738                $sensors = $device | Get-Sensor
739
740                # Save our current object
741                $deviceObj = $_
742                
743                $servicesToAdd = @()
744                # Loop through the services
745                $deviceObj.Services | ForEach-Object {
746                    # Get the new prefix
747                    $sensorPrefix = $SensorType.Value.SensorPrefix
748                    $normalizedName = $_.Name.Replace("  "," ").Replace("\\","\")
749                    # Build the new sensor name and normalize it so we don't accidently create two sensors just because one messed up at creating
750                    $newSensorname = "$sensorPrefix $normalizedName"
751
752                    # Check if the sensor already exists
753                    if(
754                        $sensors.Name -and
755                        ($newSensorname -in $sensors.Name.Replace("  "," "))
756                    ){
757                        Write-Host -ForegroundColor Green -Object "[$($deviceObj.DeviceName)]: $newSensorname exists."
758
759                    # No it doesn't -> create it
760                    }else{
761                        Write-Host -ForegroundColor Yellow -Object "[$($deviceObj.DeviceName)]: $newSensorname adding to process table.."
762                        $params = @{
763                            RawType = $SensorType.Value.RawType
764                            ErrorAction = 'Stop'
765                            Target = $_
766                        }
767                        
768                        # Check if we got additional QueryParameters and add them to our Splatting
769                        if($SensorType.Value.QueryParameters){
770                            $params.QueryParameters = $SensorType.Value.QueryParameters
771                            # Timeout requires an extra switch, if it is set, set it here also
772                            if($SensorType.Value.QueryParameters.'timeout_'){
773                                $params.Timeout = $SensorType.Value.QueryParameters.'timeout_'
774                            }
775                        }
776                        try{
777                            # Create SensorParameter Object based on device
778                            $sParam = $device | New-SensorParameters @params
779
780                            # Dynamically add the timeout once more
781                            if($params.Timeout){
782                                # Add Timeout Switch if set - there seems to be a bug in the module so it does not inherit the one we set earlier, this would generate errors for NetApp cDOT Sensors 
783                                if(
784                                    $SensorType.Value.QueryParameters -and
785                                    $SensorType.Value.QueryParameters.'timeout_'
786                                ){
787                                    # We need to unlock the object before adding custom parameters to it
788                                    $sParam.Unlock()
789                                    # Set the timeout
790                                    $sParam.timeout = $SensorType.Value.QueryParameters.'timeout_'
791                                    # Lock it again
792                                    $sParam.Lock()
793                                }
794                            }
795
796                            # Add the actual sensor to our process list
797                            $servicesToAdd += @{
798                                SensorParams = $sParam
799                                SensorType = $SensorType.Name
800                                SensorDeviceName = $device.Name
801                                SensorFriendlyName = "[$($device.Name)] $normalizedName"
802                            }
803                            Write-Host -ForegroundColor Yellow -Object "[$($deviceObj.DeviceName)]: $newSensorname added to process table."
804                        }catch{
805                            Write-Host -ForegroundColor Red -Object "[$($deviceObj.DeviceName) - $($SensorType.Name)]: $newSensorname failed to create: $_"
806                        }
807                    }        
808                }
809
810                # Process new sensors
811                if($servicesToAdd.Count -gt 0){
812                    # Check if we have to add too many sensors
813                    $MaxSensors = 10
814                    if($SensorType.Value.MaxSensors){
815                        $MaxSensors = $SensorType.Value.MaxSensors
816                    }
817
818                    if($servicesToAdd.Count -gt $MaxSensors){
819                        $lError = "[$($deviceObj.DeviceName)]: has over $MaxSensors Sensors to add, please review - skipping for now."
820                        $hasError += $lError
821                        Write-Host -ForegroundColor Red -Object $lError
822                    }else{
823                        Write-Host -ForegroundColor Yellow -Object "[$($deviceObj.DeviceName)]: Processing new Sensors.."
824                        $servicesToAdd.GetEnumerator() | ForEach-Object {
825                            Write-Host -ForegroundColor Yellow -Object "[$($deviceObj.DeviceName)]: adding $($_.SensorFriendlyName).."
826                            try{
827                                $null = $_.SensorParams | Add-Sensor -ErrorAction Stop
828                                $addedSensors += @{
829                                    name = $_.SensorType
830                                    value = $_.SensorFriendlyName
831                                }
832                                $servicesAdded += $_
833                            }catch{
834                                $lError = "[$($deviceObj.DeviceName)]: Failed adding $($_.SensorFriendlyName): $($_.InvocationInfo.ScriptLineNumber) - $($_.Exception.Message)"
835                                Write-Host -ForegroundColor Red -Object $lError
836                                $hasError += $lError
837                            }
838                        }
839                    }
840                }
841            }
842        }catch{
843            $lError = "Failed at Sensor Type: $($SensorType.Name), Error in line $($_.InvocationInfo.ScriptLineNumber): $($_.Exception.Message)"
844            Write-Host -ForegroundColor Red -Object $lError
845            $hasError += $lError
846        }    
847    }
848}catch{
849    $lError = "Failed to discover! Error in line $($_.InvocationInfo.ScriptLineNumber): $($_.Exception.Message)"
850    Write-Host -ForegroundColor Red -Object $lError
851    $hasError += $lError
852}finally{
853    if(
854        $addedSensors -and
855        $addedSensors.Count -gt 0
856    ){
857        Write-Host -ForegroundColor Green -Object "Summary:"
858        $servicesAdded.GetEnumerator() | ForEach-Object {
859            Write-Host -ForegroundColor Green -Object "[$($_.SensorDeviceName)]: added $($_.SensorFriendlyName).."
860        }
861        if($TeamsURL){
862            Send-TeamsMessage -Facts $addedSensors
863        }
864    }
865    
866    # Cleanup orphaned Windows Service sensors, this happens from time to time during service discovery unfortunately
867    # Those sensors can be identified by their Name "Service " and their type of snmpservice - however they do not work as they don't get an OID
868    try{
869        $cleanup = Get-Sensor -Name "Service " | Where-Object { $_.Type.StringValue -eq "snmpservice" -and $_.Name -eq "Service " }
870        if($cleanup -and $cleanup.Count -gt 0){
871            Write-Host -ForegroundColor Yellow -Object "Cleaning up orphaned SNMP Service sensors.."
872            $cleanup | ForEach-Object {
873                Write-Host -ForegroundColor Yellow -Object "[$($_.Device)]: Cleaning up orphaned SNMP Service sensors.."
874                $_ | Remove-Object -Confirm:$false -Force
875            } 
876        }        
877    }catch{
878        Write-Host -ForegroundColor Yellow -Object "Error at cleaning up orphaned SNMP Service sensors! Error: $_"
879    }
880
881    if($hasError.Count -gt 0){
882        Write-Host -ForegroundColor Red -Object "Errors detected, please review the following lines!"
883        $hasError | ForEach-Object {
884            Write-Host -ForegroundColor Red -Object $_
885        }
886        exit 1;
887    }
888}
